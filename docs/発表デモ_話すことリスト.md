# 発表デモ（約4分）で話すことリスト

## 0:00–0:20 導入
- 「歌詞を GoEmotions の28次元感情ベクトルにして、曲同士の近さから推薦＋可視化するアプリです」
- 「左が操作（検索・手法・Top‑K・拡張）、右が結果（Top‑K表とネットワーク）です」

## 0:20–0:50 検索と曲選択
- 「タイトル/アーティストで絞り込んで、曲を選ぶと推薦が更新されます」
- 「まずこの曲を選びます（例：Killa Cam など）」

## 0:50–1:15 Query song
- 「選んだ曲の基本情報がここに出ます」
- 「YouTubeで確認できるので、デモ中に“実在の曲”であることが分かりやすいです」

## 1:15–1:55 Top‑K similar songs（表）
- 「上位K件の類似曲を表で出します」
- 「Kを変えると候補の広さが変わります（デモでは少し動かして見せる）」
- 「各行から YouTube を開けるので、結果の納得感をすぐ確認できます」

## 1:55–2:55 Similarity method（似ているの定義を切り替え）
- 「このアプリのポイントは、“似ている”の定義を切り替えられることです」
- 「同じ曲でも手法を変えると上位が変わるので、推薦の見え方が変わります」
- 切り替えながら言うこと（短く）
  - **raw cosine**: 基準（28次元そのままの cosine）
  - **cosine without neutral**: neutral の影響を弱める
  - **mean-centered / zscore**: 共通ベースを除いて差分を見る
  - **[]Annoy angular**: 近似検索で高速に Top‑K（規模が大きくなっても使える）
- 「“説明を表示/非表示”で要点が見られて、“詳細説明”で新タブに詳説があります」

## 2:55–3:45 Connected graph（探索できる推薦）
- 「表だけだと関係が見えにくいので、Connected Papers 風に近傍をグラフで可視化します」
- 「ノードをクリックすると詳細（曲名/アーティスト/YouTube）が見られます」
- 「“拡張する”で、その曲の近傍を追加して探索できます」
- 「Reset graph で探索をリセットできます（デモでやり直しがしやすい）」

## 3:45–4:00 まとめ
- 「同じデータでも手法で近傍が変わる → 納得感や多様性が変わる」
- 「表とグラフで“探索できる推薦”にしたのがポイントです」
